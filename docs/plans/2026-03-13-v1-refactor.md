# haskell-agentic v1 Refactor Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Refactor haskell-agentic from a prototype using `MonadRWS` + `Kleisli` arrows to a v1 library using `effectful-core` discrete effects, plain functions as agents, and a clean module namespace ready for Hackage release.

**Architecture:** Replace `AgenticRWS m = MonadRWS Environment Events State m` with discrete `effectful` effects (`LLM`, `AgentConfig`, `AgentEvents`, `AgentSession`). Replace `Kleisli` arrows with plain `Agent es a b = a -> Eff es b`. Keep Dhall as the default schema format, JSON Schema as secondary. Provider interpreters (`runAnthropic`, etc.) close over their config and live in `Agentic.Provider.*`.

**Tech Stack:** `effectful-core` (effects), `dhall` (default schema), `aeson` + `autodocodec-schema` (JSON Schema), `http-conduit` (Anthropic provider), GHC2024, Cabal 3.0

```haskell
  | ToolInputDecodeFailed SchemaError
  | ToolExecutionFailed e
```

```haskell
module Agentic.Core
  ( Agent
  , fanoutMap
  , sequential
  ) where

import Effectful (Eff)
import UnliftIO.Async (mapConcurrently)

-- | An agent is a plain effectful function.
type Agent es a b = a -> Eff es b

-- | Map an agent over a list, running each in parallel.
fanoutMap :: Agent es a b -> Agent es [a] [b]
fanoutMap agent inputs = mapConcurrently agent inputs

-- | Sequential composition: feed output of first agent into second.
-- Use (>=>) from Control.Monad for inline composition.
sequential :: Agent es a b -> Agent es b c -> Agent es a c
sequential f g = \a -> f a >>= g
```

```haskell
module Agentic.Schema.Dhall
  ( dhallSchemaOf
  , parseDhall
  , dhallSystemPrompt
  , injectDhallSchema
  , injectDhallObject
  , dhallRetryPrompt
  ) where

import Agentic.Error (SchemaError(..))
import Control.Exception (SomeException, try)
import Data.Either.Validation (Validation(..))
import Data.Proxy (Proxy(..))
import Data.Text (Text, pack)
import Dhall (FromDhall, ToDhall)
import qualified Dhall
import qualified Dhall.Core

-- | Render the Dhall type schema for a type.
dhallSchemaOf :: forall a. (FromDhall a, ToDhall a) => Proxy a -> Text
dhallSchemaOf _ = case Dhall.expected (Dhall.auto @a) of
  Success result -> Dhall.Core.pretty result
  Failure err    -> error $ show err  -- schema derivation is total for Generic types

-- | Parse a Dhall text value into @a@, returning a typed error.
parseDhall :: forall a. FromDhall a => Text -> IO (Either SchemaError a)
parseDhall input = do
  result <- try @SomeException $ Dhall.input Dhall.auto input
  pure $ case result of
    Right value -> Right value
    Left err    -> Left $ DhallParseError $ pack $ show err

-- | Inject the Dhall schema for @a@ into a user prompt.
injectDhallSchema :: forall a. (FromDhall a, ToDhall a) => Proxy a -> Text -> Text
injectDhallSchema proxy prompt =
  prompt <> "\n\nReturn response in Dhall format using the following schema:\n" <> dhallSchemaOf proxy

-- | Inject a Dhall-serialised value into a prompt.
injectDhallObject :: forall a. ToDhall a => a -> Text -> Text
injectDhallObject obj prompt =
  let dhallValue = Dhall.Core.pretty $ Dhall.embed Dhall.inject obj
  in prompt <> "\n\nInput:\n" <> dhallValue

dhallRetryPrompt :: Text -> Text -> Text -> Text
dhallRetryPrompt err reply originalInstruction =
  "Your last reply failed Dhall parsing with the following error:\n" <> err
  <> "\n\nYour response was:\n" <> reply
  <> "\n\nThe instruction you were given was:\n" <> originalInstruction
  <> "\n\nPlease fix the problem and respond with the correct output"

-- Reused verbatim from the prototype.

dhallSystemPrompt = -- (paste the languageReference1_no_functions text from Protocol/DhallSchema/Prompts.hs)
```

```haskell
module Agentic.Schema.Json
  ( jsonSchemaOf
  , parseJson
  , injectJsonSchema
  , injectJsonObject
  ) where

import Agentic.Error (SchemaError(..))
import Autodocodec (HasCodec)
import Autodocodec.Schema (jsonSchemaViaCodec)
import Data.Aeson (FromJSON, ToJSON, eitherDecodeStrict, encode)
import Data.Aeson.Encode.Pretty (encodePretty)
import Data.ByteString.Lazy (toStrict)
import Data.Text (Text, pack)
import Data.Text.Encoding (decodeUtf8, encodeUtf8)

-- | Render the JSON Schema for a type as Text.
jsonSchemaOf :: forall a. HasCodec a => Text
jsonSchemaOf = decodeUtf8 $ toStrict $ encodePretty $ jsonSchemaViaCodec @a

-- | Parse a JSON-encoded value, returning a typed SchemaError.
parseJson :: forall a. FromJSON a => Text -> Either SchemaError a
parseJson input = case eitherDecodeStrict (encodeUtf8 input) of
  Right value -> Right value
  Left err    -> Left $ JsonParseError "" (pack err)

-- | Inject the JSON Schema for @a@ into a user prompt.
injectJsonSchema :: forall a. HasCodec a => Text -> Text
injectJsonSchema prompt =
  prompt <> "\n\nRespond with valid JSON matching this schema:\n" <> jsonSchemaOf @a

-- | Inject a JSON-serialised value into a prompt.
injectJsonObject :: forall a. ToJSON a => a -> Text -> Text
injectJsonObject obj prompt =
  prompt <> "\n\nInput:\n" <> decodeUtf8 (toStrict (encodePretty obj))
```

### Task 8: Implement prompt + extract in `Agentic.Prompt`

**Files:**
- Create: `src/Agentic/Prompt.hs`

**Step 1: Write the module**

This is the heart of the v1 API. `prompt` builds an LLM call, `extract` parses the response using the default (Dhall) format, `extractWith` dispatches on `SchemaFormat`.

```haskell
module Agentic.Prompt
  ( prompt
  , extract
  , extractDhall
  , extractJson
  , extractWith
  , inject
  , injectJson
  ) where

import Agentic.Effects
  ( LLM(..), LLMMessage(..), LLMRequest(..), LLMResponse(..)
  , AgentEvents(..)
  , call, emit
  )
import Agentic.Error (SchemaError)
import Agentic.Retry (withRetry)
import Agentic.Schema (SchemaFormat(..))
import Agentic.Schema.Dhall (dhallSchemaOf, parseDhall, injectDhallSchema, injectDhallObject, dhallSystemPrompt)
import Agentic.Schema.Json (jsonSchemaOf, parseJson, injectJsonSchema, injectJsonObject)
import Autodocodec (HasCodec)
import Data.Aeson (FromJSON, ToJSON)
import Data.Proxy (Proxy(..))
import Data.Text (Text)
import Dhall (FromDhall, ToDhall)
import Effectful (Eff, (:>))
import Effectful.Error.Static (Error, throwError)

-- | Send a text prompt to the LLM and return the raw response text.
prompt :: LLM :> es => Text -> Eff es Text
prompt userMsg = do
  let req = LLMRequest
        { messages     = [LLMMessage { role = "user", content = userMsg }]
        , tools        = []
        , systemPrompt = Just dhallSystemPrompt
        }
  resp <- call req
  case resp.content of
    Just txt -> pure txt
    Nothing  -> pure ""

-- | Extract a typed value using the Dhall schema format (default).
extract :: forall a es.
  ( FromDhall a, ToDhall a
  , LLM :> es
  , AgentEvents :> es
  , Error SchemaError :> es
  )
  => Text -> Eff es a
extract = extractDhall

-- | Extract using Dhall explicitly.
extractDhall :: forall a es.
  ( FromDhall a, ToDhall a
  , LLM :> es
  , AgentEvents :> es
  , Error SchemaError :> es
  )
  => Text -> Eff es a
extractDhall userMsg = do
  let schemaPrompt = injectDhallSchema (Proxy @a) userMsg
      req = LLMRequest
        { messages     = [LLMMessage { role = "user", content = schemaPrompt }]
        , tools        = []
        , systemPrompt = Just dhallSystemPrompt
        }
  withRetry 3 req $ \responseText -> do
    result <- liftIO $ parseDhall @a responseText
    pure result

-- | Extract using JSON Schema explicitly.
extractJson :: forall a es.
  ( FromJSON a, ToJSON a, HasCodec a
  , LLM :> es
  , AgentEvents :> es
  , Error SchemaError :> es
  )
  => Text -> Eff es a
extractJson userMsg = do
  let schemaPrompt = injectJsonSchema @a userMsg
      req = LLMRequest
        { messages     = [LLMMessage { role = "user", content = schemaPrompt }]
        , tools        = []
        , systemPrompt = Nothing
        }
  withRetry 3 req $ \responseText ->
    pure $ parseJson @a responseText

-- | Extract using an explicit SchemaFormat value.
extractWith :: forall a es.
  ( LLM :> es
  , AgentEvents :> es
  , Error SchemaError :> es
  )
  => SchemaFormat a -> Text -> Eff es a
extractWith Dhall msg = extractDhall msg
extractWith Json  msg = extractJson  msg

-- | Inject a Dhall-serialisable value into a prompt.
inject :: forall a. ToDhall a => a -> Text -> Text
inject = injectDhallObject

-- | Inject a JSON-serialisable value into a prompt.
injectJson :: forall a. (ToJSON a, HasCodec a) => a -> Text -> Text
injectJson = injectJsonObject
```

### Task 9: Implement retry logic in `Agentic.Retry`

**Files:**
- Create: `src/Agentic/Retry.hs`

**Step 1: Write the module**

The retry loop: sends an LLM request, tries to parse the response, on failure appends the error to the conversation and retries up to N times.

```haskell
module Agentic.Retry
  ( withRetry
  ) where

import Agentic.Effects
  ( LLM(..), LLMMessage(..), LLMRequest(..), LLMResponse(..)
  , AgentEvents(..)
  , Event(..)
  , call, emit
  )
import Agentic.Error (SchemaError(..))
import Agentic.Schema.Dhall (dhallRetryPrompt)
import Data.Text (Text, pack)
import Effectful (Eff, (:>))
import Effectful.Error.Static (Error, throwError)
import GHC.Natural (Natural)

-- | Run an LLM request, parse the response, and retry on parse failure.
-- On each retry the parse error and original bad response are fed back to the LLM.
withRetry
  :: ( LLM :> es
     , AgentEvents :> es
     , Error SchemaError :> es
     )
  => Natural                          -- ^ Maximum attempts
  -> LLMRequest                       -- ^ Initial request
  -> (Text -> Eff es (Either SchemaError a))  -- ^ Parser
  -> Eff es a
withRetry maxAttempts initialReq parser = go 1 initialReq
  where
    go attempt req
      | attempt > maxAttempts = do
          emit $ RetryAttempt attempt maxAttempts (DhallParseError "max retries exceeded")
          throwError (DhallParseError "extraction failed after max retries")
      | otherwise = do
          emit $ SchemaExtractAttempt attempt DhallFormat (error "TypeRep not threaded yet")
          resp <- call req
          case resp.content of
            Nothing -> throwError (DhallParseError "LLM returned no content")
            Just responseText -> do
              result <- parser responseText
              case result of
                Right value -> do
                  emit $ SchemaExtractSuccess DhallFormat (error "TypeRep not threaded yet")
                  pure value
                Left err -> do
                  emit $ RetryAttempt attempt maxAttempts err
                  let errorMsg = renderSchemaError err
                      -- Append error context to the conversation as a new user message
                      retryUserMsg = dhallRetryPrompt errorMsg responseText
                                      (lastUserMessage req)
                      retryReq = req { messages = req.messages
                                        ++ [ LLMMessage "assistant" responseText
                                           , LLMMessage "user" retryUserMsg
                                           ] }
                  go (attempt + 1) retryReq

lastUserMessage :: LLMRequest -> Text
lastUserMessage req =
  case [ m.content | m <- req.messages, m.role == "user" ] of
    [] -> ""
    ms -> last ms

renderSchemaError :: SchemaError -> Text
renderSchemaError (DhallParseError t) = t
renderSchemaError (DhallTypeError  t) = t
renderSchemaError (JsonParseError p m) = p <> ": " <> m
renderSchemaError (JsonSchemaViolation t) = t
```

**NOTE:** The `TypeRep` fields in events are left as `error` placeholders for now — they are observability niceties and don't block the core retry loop. They will be threaded properly in Phase 4.

```haskell
data AnthropicConfig = AnthropicConfig

  , model  :: Text
  } deriving (Show)

defaultAnthropicConfig = AnthropicConfig

  , model  = "claude-sonnet-4-20250514"

-- | Interpret the LLM effect using the Anthropic Messages API.
runAnthropic
  :: IOE :> es
  => AnthropicConfig
  -> Eff (LLM : es) a
  -> Eff es a
runAnthropic config = interpret $ \_ (Call req) -> do
  key <- case config.apiKey of
    Just k  -> pure (T.unpack k)
    Nothing -> liftIO $ getEnv "ANTHROPIC_KEY"
  let anthropicReq = toAnthropicRequest config.model req
  resp <- liftIO $ Anthropic.messages anthropicReq
  pure $ fromAnthropicResponse resp

-- | Translate generic LLMRequest to Anthropic wire format.
toAnthropicRequest :: Text -> LLMRequest -> AT.MessagesRequest
toAnthropicRequest model' req = AT.MessagesRequest
  { AT.model         = model'
  , AT.messages      = map toAnthropicMessage req.messages
  , AT.maxTokens     = 10240
  , AT.system        = fmap (\sp -> [AT.SystemPrompt sp Nothing]) req.systemPrompt
  , AT.temperature   = Nothing
  , AT.tools         = Nothing  -- tools wired up in Task 13
  , AT.toolChoice    = Nothing
  , AT.stopSequences = Nothing
  , AT.stream        = Nothing
  , AT.metadata      = Nothing
  }

toAnthropicMessage :: LLMMessage -> AT.Message
toAnthropicMessage msg = AT.Message
  { AT.role    = msg.role
  , AT.content = [AT.TextContent msg.content]
  }

-- | Translate Anthropic response to generic LLMResponse.
fromAnthropicResponse :: AT.MessagesResponse -> LLMResponse
fromAnthropicResponse resp = LLMResponse
  { content   = firstText resp.content
  , toolCalls = []  -- tool call parsing wired up in Task 13
  , usage     = Just $ LLMUsage
      { inputTokens  = resp.usage.inputTokens
      , outputTokens = resp.usage.outputTokens
      }
  }

firstText :: [AT.ResponseContentBlock] -> Maybe Text
firstText [] = Nothing
firstText (AT.ResponseTextContent t : _) = Just t
firstText (_ : rest) = firstText rest
```

```haskell
module Agentic.Tools
  ( Tool(..)
  , ToolName(..)
  , mkTool
  , dispatchTool
  ) where

import Agentic.Error (ToolError(..), SchemaError)
import Agentic.Schema.Dhall (dhallSchemaOf, parseDhall)
import Data.Proxy (Proxy(..))
import Data.Text (Text)
import Dhall (FromDhall, ToDhall)
import qualified Dhall.Core
import qualified Dhall
import Effectful (Eff, (:>))
import Effectful.Error.Static (Error, throwError)

newtype ToolName = ToolName Text
  deriving (Eq, Ord, Show)

-- | Fully typed tool. Existential hides input/output types.
data Tool es where
  MkTool
    :: (FromDhall i, ToDhall i, ToDhall o, FromDhall o)
    => { toolName        :: ToolName
       , toolDescription :: Text
       , toolInputSchema :: Text         -- pre-derived at construction time
       , toolExecute     :: i -> Eff es o
       }
    -> Tool es

-- | Smart constructor: derives the input schema from the type.
mkTool
  :: forall i o es.
     (FromDhall i, ToDhall i, ToDhall o, FromDhall o)
  => ToolName
  -> Text
  -> (i -> Eff es o)
  -> Tool es
mkTool name desc execute = MkTool
  { toolName        = name
  , toolDescription = desc
  , toolInputSchema = dhallSchemaOf (Proxy @i)
  , toolExecute     = execute
  }

-- | Dispatch a tool call: decode input, execute, encode output.
-- Returns the encoded Dhall output as Text.
dispatchTool
  :: ( Error (ToolError Text) :> es )
  => [Tool es]
  -> Text          -- ^ ToolName
  -> Text          -- ^ Raw tool input (Dhall-encoded)
  -> Eff es Text   -- ^ Dhall-encoded output
dispatchTool tools nameText rawInput =
  case filter (\(MkTool { toolName = ToolName n }) -> n == nameText) tools of
    [] -> throwError $ ToolNotFound nameText
    (MkTool { toolExecute } : _) -> do
      input <- liftIO (parseDhall rawInput) >>= \case
        Left err    -> throwError $ ToolInputDecodeFailed err
        Right value -> pure value
      output <- toolExecute input
      pure $ Dhall.Core.pretty $ Dhall.embed Dhall.inject output
```

```haskell
  deriving (Generic, Show, FromDhall, ToDhall)

data DinoPic = DinoPic { asciiPic :: Text }
  deriving (Generic, Show, FromDhall, ToDhall)

  { size :: Int, ferocity :: Int, speed :: Int, intelligence :: Int, armor :: Int }
  deriving (Generic, Show, FromDhall, ToDhall)

drawPic :: LLM :> es => Agent es Text DinoPic
```

## Common Pitfalls

1. **`effectful-core` vs `effectful`**: `effectful-core` exports `Eff`, `:>`, `Effect`, `IOE`, pure effects (`State`, `Reader`, `Error`). It does NOT export `IOE`. Use `Effectful.IOE` from `effectful` (full package) or handle IO via `liftIO` with `IOE :> es`. Re-check which package exports what if imports fail.

2. **`makeEffect ''LLM`**: Requires `TemplateHaskell` extension and generates `call :: LLM :> es => LLMRequest -> Eff es LLMResponse`. The TH splice must come AFTER the data definition.

3. **DuplicateRecordFields**: Both `LLMRequest` and `LLMResponse` have a `content` field. Use `-XOverloadedRecordDot` and explicit type annotations when there's ambiguity.

4. **Dhall `embed`/`inject`**: `Dhall.inject` is the `Encoder`, `Dhall.embed Dhall.inject obj` produces a `Dhall.Expr`. `Dhall.Core.pretty` renders it as Text.

5. **`IOE :> es` for `liftIO`**: Any `Eff es` computation that calls `liftIO` needs `IOE :> es` in its constraint, which is provided by `runEff` at the top level.
